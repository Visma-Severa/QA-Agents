## CONTENT FILTERING PRIORITY

When reducing word count, remove in this order:

1. **Keep (Never Remove):**
   - Critical issues (all)
   - File:line references for critical issues
   - Risk badge + recommendation
   - Failed hotfix patterns

2. **Simplify First:**
   - Test coverage: Use counts only, no file names
   - Action items: Reduce to 3-5 most important
   - Hotfix patterns: Remove warnings, keep only failures

3. **Remove If Needed:**
   - High-severity issues (keep critical only)
   - Test coverage section (if no tests missing)
   - Hotfix pattern table (if all pass)
