# AI Code Review QA Agent

Analyzes branches for code quality, potential risks, test coverage gaps, and generates actionable review reports for the HealthBridge health management platform.

## Files in This Folder

| File | Purpose |
|------|---------|
| `code-review-qa.md` | Main prompt with analysis instructions, hotfix patterns, detection rules |
| `code-review-template.md` | **Report template (COPY EXACTLY)** - Defines exact section structure |
| `code-review-brief-template.md` | Brief 450-word GitHub PR comment format |
| `findings-detailed-template.md` | Deep analysis structure for flagged findings |
| `README.md` | This documentation |

## Purpose

This agent helps reviewers and QA by automatically analyzing code changes to identify:
- Code quality issues
- Potential bugs or security concerns
- **Client-Server Security Consistency** - Detects incomplete security implementations
- Performance implications
- Test coverage gaps with detailed recommendations
- E2E automation impact (Playwright, Mobile)
- Regression testing impact areas
- Historical hotfix pattern matches (Authorization gaps, NULL handling, edge cases)

## Outputs

1. **Code Review Report** - Concise review (max 1300 words) in `reports/code-review/`

**Note:** Acceptance test scenarios are generated by a separate agent (`@hb-qa-acceptance-tests`).

Copy the prompt from `code-review-qa.md` and provide it to your AI assistant along with the PR diff.

## Prompt Variables

| Variable | Description |
|----------|-------------|
| `{{PR_DIFF}}` | The full diff of the branch |
| `{{PR_TITLE}}` | Title/name of the branch or PR |
| `{{PR_DESCRIPTION}}` | Description provided by the author |
| `{{CHANGED_FILES}}` | List of files changed |

## Security Consistency Check

### What It Detects

Automatically triggers when PR modifies:
- JavaScript/TypeScript security code: tokens, authentication, session handling
- Files: `auth.ts`, `security.ts`, authentication modules
- Methods: token generation/validation, session management, permission checks

### What It Checks

1. **Client-Server Symmetry** - Does server code match client-side security changes?
2. **Dependency Impact** - Runs grep to find ALL dependent files
3. **Security Documentation** - Is WHY documented? What protection is affected?
4. **Testing Evidence** - Manual testing proof for critical security flows

### Prevention Impact

- Prevents client-server security mismatches
- Catches incomplete authentication changes
- Automatic detection - no developer action required
